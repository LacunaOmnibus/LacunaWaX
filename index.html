    INSTALLING FOR THE FIRST TIME

    • To install on Windows, download the EXE installer and run it.
    • To install on anything other than Windows, you'll need to run from source, which takes a bit more work. Reasonable installation docs for running from source are on the TODO list.

    UPGRADING FROM PREVIOUS VERSIONS

    The upgrade process has changed for version 2.0; please read below!

    • The v2.0 installer does not transfer preferences from your previous installation automatically anymore. But it does allow you to import preferences from a previous install.
    • Before running the v2.0 installer, you'll want to back up your current preferences database first.

    • To back up version 1.x
      • Shut down LacunaWaX if it's running.
      • Find your current LacunaWaX database file. This is simply a single file called lacuna_app.sqlite. You'll find it under your installation folder, something like C:\Program Files\TMT Tools\LacunaWaX\user\lacunawax_app.sqlite - copy that file onto your desktop.
    • To back up version 2.x
      • Start LacunaWax. Under the File menu, choose "Export Database". Browse to your desktop, and save the database there.

    • Uninstall LacunaWaX as you'd uninstall any other program (Control Panel... Add/Remove Programs).
    • Install the new version of LacunaWaX. When the installation completes, run it.
    • Choose the File... Import Preferences menu item, and browse to the lacunawax_app.sqlite file you created in the "back up" step above. This will import your old preferences to your new installation.
    • You can now run LacunaWaX normally, and you're free to delete the lacunawax_app.sqlite file from your desktop.

    CAUTION

    LacunaWaX stores game account passwords in plain text. Guard your lacuna_app.sqlite file - handing it to someone else for any reason is the same as handing that person not only your own account password, but also all of your recorded sitter passwords.

    It is safe to send your lacuna_log.sqlite file to someone else for debugging help if needed, as it contains no passwords.

    RUNNING FROM SOURCE

    Quite a few non-core Perl modules are required, and no make process exists yet, so pre-requisite modules will need to be installed manually.

    REQUIREMENTS

    This list is not yet exhaustive.

    • Perl >= v5.14
    • Archive::Zip
    • CHI
    • DateTime
    • JSON::RPC::LWP
      • You may need to force install this. cpan -f JSON::RPC::LWP
    • LWP::UserAgent
    • Moose
    • MooseX::NonMoose
    • Try::Tiny
    • URI
    • Wx

    Run

     $ perl bin/LacunaWaX.pl

    Install any modules it complains about missing, and add them to the list above.
    Once all pre-reqs have been installed, that command will start the GUI.
